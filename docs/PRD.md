# NexusNote PRD

更新时间：2026-04-01

## 产品定位

NexusNote 是一款 AI-native 学习应用，不是通用笔记工具，也不是单纯聊天壳。

它的主链是：

`学习目标 -> 课程访谈 -> 课程生成 -> 章节学习 -> 学习对话 -> 知识沉淀 -> 技能/路线成长`

产品重点不是“让用户写更多内容”，而是让用户围绕真实学习目标持续推进，并把过程沉淀成可复用知识。

## 目标用户

- 有明确学习目标、希望快速获得结构化学习路径的个人用户
- 希望把课程学习、AI 对话、笔记、高亮、沉淀放进同一工作流的重度学习者
- 希望围绕技能成长和职业路线持续积累的用户

当前不是面向：

- 纯团队协作文档平台
- 通用离线优先知识库
- 大型企业知识管理系统

## 核心价值

### 1. 从目标直接生成可学的课程

用户不需要先自己搭框架。

通过访谈，系统直接输出接近真实课程的完整草案，并能进一步生成章节内容。

### 2. 学习时 AI 真正理解课程上下文

学习对话不是脱离上下文的通用聊天，而是可以围绕当前课程、当前章节、当前学习进度展开。

### 3. 学习过程自动沉淀为知识资产

高亮、笔记、对话沉淀、课程进度不是孤岛，而是共同形成可检索、可回顾、可复用的知识工作台。

### 4. 学习结果映射到技能与成长路线

课程和章节不仅是内容结构，也要能映射到技能节点和成长路径，帮助用户理解“我正在走向哪里”。

## 当前产品模块

### 课程访谈

- 通过多轮访谈澄清学习目标、基础、应用场景和约束
- 输出课程草案预览
- 支持在草案上继续修改，而不是每次重新从头访谈

### 课程生成与学习

- 生成课程大纲与章节内容
- 学习页展示课程结构、章节内容、学习对话和进度
- 支持章节级高亮与注释

### 学习对话

- 支持围绕学习上下文的流式对话
- 区分普通对话与课程内对话
- 服务端维护上下文、工具调用与流式输出契约

### 笔记与沉淀

- 用户可捕获课程高亮、课程笔记和学习对话
- 沉淀内容进入笔记系统，形成后续检索与回顾基础
- 笔记系统目标是跨课程知识工作台，而不是单文档编辑器集合

### 技能与黄金之路

- 课程和章节映射到技能点
- 通过课程进度驱动技能路线进度
- 为后续成长路径和职业路线提供结构化基础

## 当前设计原则

- 课程优先，不做泛化成“什么都能装”的工作台
- AI 结果必须服务真实 UI 和真实流程，不做伪结构化数据
- 流式体验必须是真流式，不接受假流式占位方案
- 架构优先收敛，不为了兼容历史方案保留多套并行实现
- 文档只保留仍然指导实现的活文档，历史资料归档

## 质量要求

### AI

- AI SDK v6 原生模式
- 工具展示契约清晰隔离
- 单 provider 运行时，显式降级
- prompt 资源外置，运行时上下文由代码拼装

### Web

- Next.js 16 + Cache Components
- 局部动态边界，不允许根级 blank fallback
- 默认动态，局部 cache/tag 精细化控制

### 数据与检索

- Drizzle schema 模块化
- RAG 路径可观测
- 向量索引、全文索引、最近数据索引与查询路径一致

## 当前非目标

- 仓库内维护 Helm / ArgoCD / Flux
- 多 provider 自动切换链路
- 大而全的离线优先本地数据库体验
- 历史设计稿长期混在主 docs 入口

## 下一阶段重点

- 继续完善黄金之路的课程驱动技能映射
- 增加更系统的自动化测试和关键链路观测
- 继续收敛仓库结构和运行时实现，避免 legacy 路径回流
